YOLOv9: Learning What You Want to Learn Using Programmable Gradient Information

Preface

​ YOLO(You Only Look Once)作为目标检测领域的经典模型，他因模型轻量、实现了速度和精度和良好平衡而出名。自提出之日起，截至目前已经迭代了十个版本。

​ 因为做科创项目的缘故，我接触YOLO比较早，但是由于YOLOv5和YOLOv8这些主流稳定的YOLO模型的团队并没有发表论文，只有github仓库和说明文档，因此没有办法做文献阅读笔记。

​ 而YOLOv9的提出，是今年2月底刚刚发表在arXiv的预印版论文，这也给了我一个机会深入了解YOLOv9的创新点，同时完成本学期发展前沿这门课的大作业的一部分。

​ 那么接下来，我将从Title、Abstract、Problm Statement、Methodology、Experiments和Reference几个方面来阅读和理解这篇论文。

Title

​ YOLOv9：使用可编程梯度信息学习您想学习的内容

​ 这个题目看起来非常宏大且唬人，Programmable Gradient Information (PGI) 这个名字乍一看，会让人往底层反向传播的设计那里想，还以为是在底层进行了创新，但看完文章内容就会知道，PGI实际上就是一个辅助的(auxiliary)框架，是为了让网络的浅层也能在反向传播学习的过程中得到更多“一手的”、“深层的”信息，而不至于得到N手信息导致有效信息丢失。并且auxiliary的框架只在训练学习时使用，在推理时可以不使用，因此他不增加额外的计算开销。

​ “Finally, since auxiliary reversible branch can be removed during the inference phase, the inference capabilities of the original network can be retained. We can also choose any reversible architectures in PGI to play the role of auxiliary reversible branch.”

Abstract

​ 摘要中主要总结了目前研究存在的问题，以及本篇文章针对这些问题所做出的工作，以及实验结果可以被概括为以下四点：

(1) 现有方法忽略了一个事实，即当输入数据经过逐层特征提取和空间变换时，大量信息将会丢失

​ “Existing methods ignore a fact that when input data undergoes layer-by-layer feature extraction and spatial transformation, large amount of information will be lost.”

(2) 我们提出了可编程梯度信息(PGI)的概念来应对深度网络实现多个目标所需的各种变化。PGI可以为目标任务计算目标函数提供完整的输入信息，从而获得可靠的梯度信息来更新网络权值

​ “We proposed the concept of programmable gradient information (PGI) to cope with the various changes required by deep networks to achieve multiple objectives. PGI can provide complete input information for the target task to calculate objective function, so that reliable gradient information can be obtained to update network weights.”

(3) 此外，还设计了一种基于梯度路径规划的新型轻量级网络架构——通用高效层聚合网络（GELAN）

​ “In addition, a new lightweight network architecture – Generalized Efficient Layer Aggregation Network (GELAN), based on gradient path planning is designed.”

(4) 我们在基于 MS COCO 数据集的目标检测上验证了所提出的 GELAN 和 PGI。 结果表明，与基于深度卷积开发的最先进方法相比，GELAN 仅使用传统的卷积算子即可实现更好的参数利用率。 PGI 可用于从轻型到大型的各种模型。 它可以用来获取完整的信息，使得从头开始训练的模型能够比使用大数据集预训练的模型获得更好的结果，对比结果如图所示

​

​ 实际上，我对上述(4)中“使得从头开始训练的模型能够比使用大数据集预训练的模型获得更好的结果”这句话是存疑的，或许作者想要表达的是，由于有了auxiliary的框架，在训练时，是不能使用原有结构的预训练权重的，而这种利用PGI的框架从零开始的训练结果会好于使用预训练权重（而没有使用PGI）的训练结果。但是我认为这个结果并不具备普适性，有些夸大了。

Problem Statement

​ 这一部分对应上摘要中的第一点，要详细介绍目前研究领域中存在的难以解释的问题，也是本文研究的出发点。

现存问题：人们经常将模型收敛的困难归因于梯度消失或梯度爆炸，但现代神经网络已经通过设计归一化和激活函数从根本上解决了这个问题，但收敛速度慢或收敛困难的问题仍然存在。

原因以及解决方案的出发点：信息瓶颈(IB)，即原本来自很深网络的每经过一层网络进行变换时都会造成信息丢失，如式(1)所示

验证原因：作者将不同架构的深度网络前馈了初始权重，然后将其可视化，如图。显然，PlainNet丢失了深层物体检测所需的大量重要信息。至于ResNet、CSPNet、GELAN能够保留重要信息的比例，确实与训练后能够获得的准确率呈正相关。

现有解决方案和不足：

(1)使用可逆架构和重复的输入数据，并以显式的方式维护输入数据的信息。但这种方式会显著增加推理成本，并且输入数据层到输出层不能有太深的路径。(RevCol)

(2)使用masked建模，主要利用重构损失，采用隐式的方式最大化提取特征并保留输入信息。但重建损失有时会与目标损失冲突，而且大多数掩码机制还会产生与数据的不正确关联。

(3)引入深度监督，它利用浅层特征来预先建立从特征到目标的映射，以确保重要信息能传递到更深的层次。但它会产生误差积累。(Deep Supervision)

我的理解：

处于较深的层，他们距离输出层（或者可以理解为标签或者标准答案）近，在反向传播时，能获得最精准的要学习的信息，而距离输出较远的浅层，他们得到的信息来自于一层一层反向传播而来，在这个过程中经过各种变换，许多重要信息已经丢失。

可以类比为上中学时，坐在前排距离老师（标签）近的同学（深层），他们上课听的最清楚，知识点也更容易理解；而坐在后排的同学（浅层），就只能通过前排同学一排一排传话传过来才能学习，这样的N手知识就使得后排同学学习起来很吃力（收敛也就困难或者慢）。

Methodology

PGI

​ PGI包括三个组成部分，main branch主分支、auxiliary reversible branch辅助可逆分支以及multi-level auxiliary information多层辅助信息。

​ 主分支：可以理解为原来YOLO的主干，用于推理的架构

​ 辅助可逆分支：用于生成可靠的梯度，为主分支提供

​ 多层辅助信息：控制主分支学习可规划的多层语义信息

我们来看图：

​

​ 图中蓝色的部分就是网络的主分支；

​ 灰色的部分是辅助可逆分支，用于为主分支的浅层也提供可靠的梯度信息，在推理时可以不使用；

​ 用红色框圈起来的部分就是多层辅助信息，它的主要作用类似于一个特征金字塔，是做特征融合的，是为了让从三个检测头出来的梯度信息不那么独立。我们直到，YOLO模型有三个检测头，P3 P4 P5，由他们反向传播的梯度信息是相对独立的，分别用于处理大中小型目标。而多层辅助信息的作用就是借助了CBNet的思想，对多层的信息进行了一个特征融合再分发，解决信息独立的问题。

​ GELAN的设计主要参考了CSPNet和YOLOv7中的经典结构ELAN。

Experiments

​ 与轻量级和中型模型YOLO MS相比，YOLOv9的参数减少了约10%，计算量减少了5∼15%，AP仍然有0.4∼0.6%的提升。与YOLOv7 AF相比，YOLOv9-C的参数减少了42%，计算量减少了22%，但达到了相同的AP（53%）。与YOLOv8-X相比，YOLOv9-E参数减少16%，计算量减少27%，AP显着提升1.7%。

​ 就参数量而言，性能最好的大型模型是 RT-DETR 。使用传统卷积的YOLOv9在参数利用率上甚至比使用深度卷积的YOLO MS还要好。至于大型模型的参数利用率，也超过了使用ImageNet预训练模型的RT-DETR。在深度模型中，YOLOv9展示了使用PGI的巨大优势，通过准确保留和提取将数据映射到目标所需的信息，仅需要 66% 的参数，同时保持 RT-DETR-X 的精度。
