YOLOv9: Learning What You Want to Learn Using Programmable Gradient Information

Preface

​ YOLO(You Only Look Once)作为目标检测领域的经典模型，他因轻量、实现了速度和精度和良好平衡而出名。自提出之日起，到今年已经迭代了十个版本。

​ 因为做科创项目的缘故，我接触YOLO比较早，但是由于YOLOv5和YOLOv8这些主流稳定的YOLO模型的团队并没有发表论文，只有github仓库和说明文档，因此没有办法做文献阅读笔记。

​ 而YOLOv9的提出，是今年2月底刚刚发表在 的论文，这也给了我一个机会深入了解YOLOv9的创新点，同时完成发展前沿这门课的大作业的一部分。

​ 那么接下来，我将从Title、Abstract、Main Contributions、Problm Statement、Methodology、Experiments和Summary几个方面来阅读和理解这篇论文。

Title

​ 翻译就是：YOLOv9：使用可编程梯度信息学习您想学习的内容

​ 这个题目看起来非常宏大，并且非常唬人，Programmable Gradient Information (PGI) 这个名字乍一看，会让人往底层反向传播的设计那里想，还以为是在底层进行了变革，但看了文章内容就会知道，PGI实际上就是一个辅助的(auxiliary)框架，是为了让模型的浅层也能在反向传播学习的过程中得到更多“一手的”、“深层的”信息。并且auxiliary的框架只在训练学习时使用，在推理时可以不使用，因此他不增加额外的计算开销。

​ “Finally, since auxiliary reversible branch can be removed during the inference phase, the inference capabilities of the original network can be retained. We can also choose any reversible architectures in PGI to play the role of auxiliary reversible branch.”

Abstract

​ 摘要中主要总结了目前研究存在的问题，以及本篇文章针对这些问题所做出的工作，以及实验结果可以被概括为以下四点：

(1) 现有方法忽略了一个事实，即当输入数据经过逐层特征提取和空间变换时，大量信息将会丢失

​ “Existing methods ignore a fact that when input data undergoes layer-by-layer feature extraction and spatial transformation, large amount of information will be lost.”

(2) 我们提出了可编程梯度信息(PGI)的概念来应对深度网络实现多个目标所需的各种变化。PGI可以为目标任务计算目标函数提供完整的输入信息，从而获得可靠的梯度信息来更新网络权值

​ “We proposed the concept of programmable gradient information (PGI) to cope with the various changes required by deep networks to achieve multiple objectives. PGI can provide complete input information for the target task to calculate objective function, so that reliable gradient information can be obtained to update network weights.”

(3) 此外，还设计了一种基于梯度路径规划的新型轻量级网络架构——通用高效层聚合网络（GELAN）

​ “In addition, a new lightweight network architecture – Generalized Efficient Layer Aggregation Network (GELAN), based on gradient path planning is designed.”

(4) 我们在基于 MS COCO 数据集的目标检测上验证了所提出的 GELAN 和 PGI。 结果表明，与基于深度卷积开发的最先进方法相比，GELAN 仅使用传统的卷积算子即可实现更好的参数利用率。 PGI 可用于从轻型到大型的各种模型。 它可以用来获取完整的信息，使得从头开始训练的模型能够比使用大数据集预训练的模型获得更好的结果，对比结果如图所示

​

​ 实际上，我对上述(4)中“使得从头开始训练的模型能够比使用大数据集预训练的模型获得更好的结果”这句话是存疑的，或许作者想要表达的是，由于有了auxiliary的框架，在训练时，是不能使用原有结构的预训练权重的，而这种利用PGI的框架从零开始的训练结果会好于使用预训练权重（而没有使用PGI）的训练结果。但是我认为这个结果并不具备普适性，有些夸大了，在多数情况下，预训练还是一种更有效的手段。

Main Contribution

​ 主要贡献这里，作者写了四点，但实际上后三点只是对第一点的分别总结补充，所以我们这里只关注第一点即可

​ “We theoretically analyzed the existing deep neural network architecture from the perspective of reversible function, and through this process we successfully explained many phenomena that were difficult to explain in the past. We also designed PGI and auxiliary reversible branch based on this analysis and achieved excellent results.”

​ 我们从可逆函数的角度对现有的深度神经网络架构进行了理论分析，通过这个过程我们成功地解释了许多过去难以解释的现象。 我们还基于此分析设计了PGI和辅助可逆分支，并取得了优异的结果。

Problem Statement

​ 这一部分对应上摘要中的第一点，要详细介绍目前研究领域中存在的难以解释的问题

​ 作者首先说，人们经常降收敛的困难归因于梯度消失或梯度爆炸，但现代神经网络已经通过设计归一化和激活函数从根本上解决了这个问题，但收敛速度慢或收敛困难的问题仍然存在；

​ 接着，作者回归主题，说这个问题的根本原因是原本来自很深网络的初始梯度在传输后很快就丢失了实现目标所需的大量信息；

我个人的理解

​ 其实就是处于较深的层，他们距离输出层（或者可以理解为标签或者标准答案）近，在反向传播时，能获得最精准的要学习的信息，而距离输出较远的浅层，他们得到的信息来自于一层一层反向传播而来，在这个过程中经过各种变换，许多重要信息已经丢失。

​ 可以类比为上中学时，坐在前排距离老师（标准答案）近的同学（深层），他们上课听的最清楚，知识点也更容易理解；而坐在后排的同学（浅层），就只能通过前排同学一排一排传话传过来才能学习，这样的N手知识就使得后排同学学习起来很吃力（收敛也就困难或者慢）。

Methodology

PGI

​ PGI包括三个组成部分，main branch主分支、auxiliary reversible branch辅助可逆分支以及multi-level auxiliary information多层辅助信息。我们来直接看图：

​ (a)PAN存在的问题就是上述所说的信息丢失问题；(b)RevCol虽然通过在浅层增加分支来弥补信息丢失和不足的问题，但是会带来heavy cost，使得模型体量变大，推理速度变慢；(c)Deep Supervision存在的问题是虽然在浅层补充了输出层的信息，但是却是分离的（YOLO的结构最后的三个检测头分别携带的是大中小目标的信息，是分离的三个分支）。因此论文作者设计出了(d),也就是PGI的结构。

​ “In view of this, we regard reversible branch as an expansion of deep supervision branch, and then design auxiliary reversible branch, as shown in Figure 3 (d). As for the main branch deep features that would have lost important information due to information bottleneck, they will be able to receive reliable gradient information from the auxiliary reversible branch. These gradient information will drive parameter learning to assist in extracting correct and important information, and the above actions can enable the main branch to obtain features that are more effective for the target task. Moreover, the reversible architecture performs worse on shallow networks than on general networks because complex tasks require conversion in deeper networks. Our proposed method does not force the main branch to retain complete original information but updates it by generating useful gradient through the auxiliary supervision mechanism. The advantage of this design is that the proposed method can also be applied to shallower networks.”

​ 鉴于此，我们将可逆分支视为深度监督分支的扩展，然后设计辅助可逆分支，如图3（d）所示。 对于由于信息瓶颈而丢失重要信息的主分支深度特征，它们将能够从辅助可逆分支接收可靠的梯度信息。 这些梯度信息将驱动参数学习来协助提取正确且重要的信息，上述动作可以使主分支获得对目标任务更有效的特征。 此外，可逆架构在浅层网络上的表现比在一般网络上差，因为复杂的任务需要在更深的网络中进行转换。 我们提出的方法并不强迫主分支保留完整的原始信息，而是通过辅助监督机制生成有用的梯度来更新它。 这种设计的优点是所提出的方法也可以应用于较浅的网络。

GELAN

​ GELAN的设计主要参考了CSPNet和YOLOv7中的经典结构ELAN，设计出了一种兼顾轻量级、推理速度和准确性的广义高效层聚合网络GELAN。

​ 实际上，通过这张图我们可以看出，与CSPNet相比，GELAN只是参考了ELAN中“套娃”的思想，使用了多个而不是单个“any block”去做transition和最终的concatenation。

Experiments

​ 与轻量级和中型模型YOLO MS相比，YOLOv9的参数减少了约10%，计算量减少了5∼15%，AP仍然有0.4∼0.6%的提升。与YOLOv7 AF相比，YOLOv9-C的参数减少了42%，计算量减少了22%，但达到了相同的AP（53%）。与YOLOv8-X相比，YOLOv9-E参数减少16%，计算量减少27%，AP显着提升1.7%。

​ 就参数量而言，性能最好的大型模型是 RT-DETR 。使用传统卷积的YOLOv9在参数利用率上甚至比使用深度卷积的YOLO MS还要好。至于大型模型的参数利用率，也超过了使用ImageNet预训练模型的RT-DETR。在深度模型中，YOLOv9展示了使用PGI的巨大优势，通过准确保留和提取将数据映射到目标所需的信息，仅需要 66% 的参数，同时保持 RT-DETR-X 的精度。

Summary

​ YOLOv9的这篇文章通过引入PGI的辅助框架概念，一定程度上解决了前向传播过程中信息丢失的问题；

​ 对YOLOv7的ELAN结构做了一小部分改变，设计了GELAN模块（这一点某种程度上更像是凑贡献点）；

​ 相比于其他YOLO系列模型，能做到提升精度的同时减少参数量和提高参数利用率，是非常不错的突破。
