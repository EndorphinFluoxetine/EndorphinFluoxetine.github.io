高斯取整函数

昨天天津大学智能与计算学部夏令营机试中，第一题用到了向下取整函数，那个题的思路特别特别简单，但是实现起来把我恶心坏了，因为全程我都是用强制类型转换后的值和原值的差来实现的；并且在不同的象限的点由于横纵坐标的符号不同，还要分别考虑，所以代码写的又臭又长

刚刚突然想起来这事，就去搜了一下，结果……

C++的标准库中已经提供了一个floor函数，可以直接使用。这是最简单和推荐的方法，因为标准库函数通常是经过高度优化和广泛测试的

结果保留n位小数

这个也是吃了一次亏，还没长记性

上学期web上机考试js编程中出过一次，我当时就没想起来保留小数位数的函数名，痛失一道题的分

昨天天津大学智能与计算学部夏令营机试，我又没想起来，还好机试够良心够easy，第二题下面给我提示了，不然又白给

这次必须记录一下

在C++中，可以使用iomanip库中的setprecision和fixed来设置浮点数的保留位数；也可以只使用setprecision来指定数的总有效位数

在Java中，可以使用String.format或DecimalFormat类来控制浮点数的保留位数

在C语言中，可以使用printf函数的格式说明符来设置浮点数的保留位数

DFS

今天借着昨天天大第五题正好来学一下图的DFS，前面一直没学，拖到现在

小Z在处理一个特殊的最短路径问题，在无向图中有部分边在最初始时是上锁没法通过的，当他拿到位于节点k的钥匙后，这些边就可以通过了，通过一条边花费为边的权值，他想知道从节点1走到节点n的最小花费是多少。

输入：多组样例输入，第一行输入一个整数T表示样例数 对于每个样例，第一行包含三个整数n，m，k，图一共有n个结点m条边，节点编号从1到n，钥匙在节点k 接下来m行每行包含四个整数，s,t,w,v,表示有一条从s到t的边，权重为w，如果v为0表示不需要钥匙就可以通行，如果是1则表示该边需要拿到钥匙后才能通行。

输出：对于每组样例，输出一个整数表示从1到n的最短路，如果不能到达，输出-1

二分查找

让人头疼的边界条件在这个视频中被完美解决了，以后遇到二分查找，只需要考虑isBlue()函数的编写逻辑以及应该返回的是谁，就可以了：二分查找为什么总是写错？

求根号2的近似值，要求误差不超过10^-6

  • 如果希望得到尽可能接近根号2且不超过根号2的值，应该选择 return l
  • 如果希望得到尽可能接近根号2且不小于根号2的值，应该选择 return r
  • 如果希望得到尽可能接近根号2的值，而不考虑偏向于哪一侧，可以选择 return m

在大多数情况下，选择 return m 是最合适的，因为 m 是在循环中动态调整的中间值，通常在循环结束时已经非常接近根号2
